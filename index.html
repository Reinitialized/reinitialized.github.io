About me

I am a hobbyist Generalized System Administrator who has built a 3 node hyper-converged cluster using 12th gen Dell PowerEdge R720s within a homelab/homeprod environment. Each node runs Proxmox 7 with Ceph for scalable compute and storage resources, all backed by dual 10Gb SPF+ multi-mode standard DACs running in LACP round-robin for high availability and improved traffic throughput. These servers are connected to a Brocade ICX6610 switch, which handles 802.1p prioritized intra-VLAN traffic to minimize latency, followed with dual 40Gbps QSFP+ multi-mode standard DACs in LACP round-robin uplinked to my main router. The router consists of a Optiplex 7050 running OPNsense, which handles all inter-VLAN communications and network firewall to lock down and protect the environment. Wireless Access Points are UniFi 6 Lites managed by a UniFi Controller hosted in a LXC container. I have been developing this setup since mid 2018 to learn about hardware and software management.

Within this environment, I run the following VMs/Containers:

  • dc1.core.in.reinitialized.net:
    • Windows Server 2022 Datacenter Core in KVM
    • Roles/AD DS Forest Level 2016
    • Roles/DNS
    • Roles/File and Storage Services
    • Primary Domain Controller, used for authentication, DNS, and managing Windows 10 client machines with GPOs
  • dc2.core.in.reinitialized.net:
    • Windows Server 2022 Datacenter Core in KVM
    • Roles/AD DS Forest Level 2016
    • Roles/DNS
    • Roles/File and Storage Services
    • Secondary Domain Controller, see dc1
  • fs1.core.in.reinitialized.net:
    • Windows Server 2022 Datacenter Core in KVM
    • Roles/File and Storage Services
    • Used for network shares and storing data of redirected profiles, allowing for simplified data management and backup
  • agent.reinitialized.net:
    • Debian 10 in LXC
    • Runs TacticalRMM (a free open source bare bones RMM with basic patch management and Take Control)
  • unifi.in.reinitialized.net:
    • Debian 10 in LXC
    • UniFi controller for managing WAPs

Active Directory is used to manage all Windows based machines within my environment and provide centralized authentication where possible. Beyond this, I preference running RHEL deprived Linux Distros where supported followed with Debian if not. My ultimate goal with this setup is to define everything as Infrastructure as Code, allowing me to fully redeploy, update systems and configuration, and add additional services all with a single command, reducing potential human bias and version controlling my setup in case of needing to rollback. All of this is backed up using Proxmox Backup Server, which is then pushed to Wasabi S3 using Duplicati.
